详解网页地址栏图标 深究favicon在pc段跟移动端用法

By iToMY

5月 12 2014 更新日期:8月 12 2014

文章目录

当你浏览到其他网站时，或许你会发现地址栏上的图标，是不是会突然发觉，我的网站怎么不显示图标，图标该怎么制作呢？怎么修改代码才能加入图标呢。在添加网站图标时，您必须了解所谓的图标(Icon)是一种特殊的图形文件格式,它是以.ico 作为扩展名。看了网站上别人的介绍，好像只需要将这个图标文件(favicon.ico)上传到您的网站所在的服务器的根目录就行了，可事实上是这样的吗？ Favicon是favorites icon的缩写，亦被称为website icon（网页图标）、page icon（页面图标）或urlicon（URL图标）。Favicon是与某个网站或网页相关联的图标。网站设计者可以多种方式创建这种图标，而目前也有很多网页浏览器支持此功能。浏览器可以将favicon显示于浏览器的地址栏中，也可置于书签列表的网站名前，还可以放在标签式浏览界面中的页标题前。要想地址栏出现图标，可以使用HTML来为任何一个网页指示其图标所存储的位置。在页面的<head>部分添加两个link组件来实现的。这样，任何一个适当大小的（16×16像素或更大）的图像都可以用作favicon。尽管如此，多数情况下仍然使用ICO格式。其他网页浏览器现在也支持PNG（便携式网络图片）格式，和GIF（图形交换格式）动画图像格式。 包含以下两行HTML代码：

  1. <link rel=“shortcut icon” href=“http://example.com/favicon.ico“ type=“image/vnd.microsoft.icon”>
  2. <link rel=“icon” href=“http://example.com/favicon.ico“ type=“image/vnd.microsoft.icon”>

只有第一行是必须的，因为“shortcut icon”字符串将被多数遵守标准的浏览器识别为列出可能的关键词（“shortcut”将被忽略，而仅适用“icon”）；而Internet Explorer将会把它作为一个单独的名称（“shortcut icon”）。这样做的结果是所有浏览器都可以理解此代码。只有当希望为新浏览器提供另一种备用图像（例如动画GIF）时，才有必要添加第二行。而且，图标是不必放在根目录的，只是当favicon.ico被置于文档根目录时，将会被一些不处理link组件的浏览器找到，即使没有您的站点上没有指向它的链接。 如果，你在网页日志记录老提示有apple-touch-icon-precomposed.png这个文件的404呢，那是mac设备来的，自动加载这个文件了。 因此，由于ios系统及Android系统的触屏智能手机和平板电脑的出现，开始出现了一种新的icon用法 <link rel=”apple-touch-icon” href=’xx.png’> <link rel=”apple-touch-icon-precomposed” href=’xx.png’> 当我们将图片作为主屏幕图标, iOS 系统会自动为图标加上视觉效果：加高亮-圆角-阴影。所以我们无需对图片做太多处理, 提供符合尺寸的正方形图片即可。无论是将图标放在默认位置上, 还是指定图标路径, 如果图标带有 -precomposed 后缀 (如: apple-touch-icon-precomposed.png), 系统将不会为你加上高亮效果. 用途是用于在将网页保存到主屏幕时，显示的相应图标。 对于图标文件type的定义 1、ICO 文件 image/vnd.microsoft.icon （一种用于Microsoft Windows上图标的光栅格式）刚开始时没有一个注册的MIME类型，而且似乎在当时也不能被多数浏览器理解。然而2003年，这一格式在IANA获得注册，其MIME类型是image/vnd.microsoft.icon，进而消除了此问题的第一部分。包括多种分辨率（最常使用的是16×16和32×32，Mac OS X有时使用64×64和128×128）以及位深（比特每像素）（多数使用4、8、24 bpp，即16、256和1600万色）（或者亦可出于兼容性原因使用image/x-icon。然而最好使用IANA注册的MIME类型，因为多数主流浏览器现在支持它） 2、GIF 文件 image/gif 使用16×16，256色。PNG：使用16×16，256色或24位。 3、PNG 文件 image/png使用适当的分辨率和色深。 因此建议代码如下：

  1. <link rel=“shortcut icon” href=“/favicon.ico”>
  2. <link rel=“icon” href=“/favicon.ico” type=“image/x-icon”>
  3. <link rel=“bookmark” href=“/favicon.ico”>
  4. <!—在收藏夹中显示自己的图标—>
  5. <link rel=“apple-touch-icon-precomposed” sizes=“72x72” href=“/xx.png”>
  6. <!—把-precomposed删掉会加阴影，定义 sizes 属性, 不定义，默认 sizes 是 57 x 57 —>
